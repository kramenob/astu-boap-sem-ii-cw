# 2. Рабочий проект

## 2.1 Общие сведения о работе системы

Разработанная информационная система представляет собой CLI-утилиту под именем `${APP_NAME}$`, образованного от ${APP_NAME_MEANING_FROM}, предназначенную для автоматизированной генерации текстовых документов на основе табличных записей, хранящихся в базе данных, и заранее подготовленных шаблонов документов, таблиц и иных текстовых форматов.

Система реализована в виде кроссплатформенного CLI-утилиты на языке C++ и функционирует по принципу командного интерфейса, подобно таким инструментам как Git, Docker, Pandoc, SQLite, FFmpeg, CMake или cURL. Основная логика работы строится вокруг обработки пользовательских команд, взаимодействия с базой данных и генерации выходных документов.

Архитектурно система состоит из двух основных компонентов:

- подсистема обработки команд (`CommandRouter`);
- подсистема хранения и обработки данных (`Database`).

Запуск системы осуществляется через терминал с передачей команд и параметров. Пример базового запуска:

```sh
${APP_NAME}$ <command> [options]
```

Система поддерживает локальную работу без подключения к сети, что обеспечивает независимость от внешних сервисов и повышает безопасность обработки данных.

## 2.2 Функциональное назначение

Основное функциональное назначение программы заключается в автоматизации процесса подготовки текстовых документов на основе структурированных данных и шаблонов.

Система выполняет следующие ключевые функции:

- управление структурой базы данных (создание, удаление, просмотр таблиц) по готовому шаблону или самостоятельно заданной схеме таблицы;
- управление данными (просмотр и обработка записей);
- работа с шаблонами документов (добавление, удаление, просмотр);
- генерация итоговых документов на основе шаблонов и данных;
- сохранение результатов в файловую систему по стандартному или специфичному пути.

Функциональность системы реализуется через набор CLI-команд:

1. **Работа с базой данных (`db`)**

| Код   | Команда                                     | Описание                                                                        |
| ----- | ------------------------------------------- | ------------------------------------------------------------------------------- |
| DB001 | `db --init`                                 | создать таблицу в интерактивном режиме                                          |
| DB002 | `db --init --templates`                     | вывести список доступных готовых шаблонов таблиц                                |
| DB003 | `db --init --template <teachers\|students>` | инициализировать таблицу по готовому шаблону (доступны `teachers` и `students`) |
| DB004 | `db --list`                                 | вывести списка пользовательских таблиц                                          |
| DB005 | `db --list --all`                           | вывести списка всех таблиц (включая системные)                                  |
| DB006 | `db --drop <table_name>`                    | удалить таблицы под именем `<table_name>`                                       |
| DB007 | `db --drop --all`                           | удалить всех таблиц                                                             |
| DB008 | `db --schema-show <table_name>`             | вывести структуры таблицы под именем `<table_name>`                             |

## 2.3 Описание программы

Программа `${APP_NAME}$` реализована в формате модульной CLI-утилитой на языке C++, использующая принципы разделения ответственности и расширяемости [?].

Основные компоненты системы:

1. **Подсистема обработки команд (`CommandRouter`)**

Обеспечивает взаимодействие пользователя с приложением через интерфейс командной строки. Выполняет разбор аргументов командной строки и маршрутизацию запросов к соответствующим обработчикам команд.

В состав подсистемы входят следующие обработчики:

- `DbCommand` — работа со структурой базы данных;
- `DataCommand` — просмотр и обработка данных;
- `TemplateCommand` — управление шаблонами документов;
- `GenerateCommand` — генерация документов;
- `HelpCommand` — вывод справочной информации;
- `VersionCommand` — отображение сведений о версии программы.

Подсистема реализует паттерн Command (Command pattern), обеспечивающий единый механизм обработки и выполнения CLI-команд.

2. **Подсистема хранения данных (`Database`)**

Реализует lightweight embedded DB layer и обеспечивает локальное хранение структурированных данных в формате SQLite [?]. Подсистема отвечает за:

- создание таблиц;
- выполнение CRUD-операций;
- загрузку seed-данных из SQL-файлов;
- предоставление данных обработчикам команд.

3. **Служебные компоненты**

Содержат сведения о приложении (версия, описание и другие метаданные), используемые при работе системы справки и отображении служебной информации.

## 2.4 Разработанные меню и интерфейсы

Интерфейс программы реализован исключительно в консольном виде и представляет собой командную систему управления.

Главное меню фактически соответствует справке:

```
${APP_NAME}$ --help
```

Вывод содержит:

- список доступных команд;

- параметры каждой команды;

- примеры использования.

**Пример интерфейса работы:**

```
${APP_NAME}$ db --init --template students
${APP_NAME}$ template --show letter
${APP_NAME}$ generate --source students --template letter
```

Интерфейс построен по принципу Unix-like CLI утилит и напоминает работу инструментов:

- Git (по структуре команд)

- Pandoc (по логике обработки вход/выход)

- SQLite (по взаимодействию с локальной БД)

**Особенности интерфейса:**

- отсутствие графического UI;

- строго текстовый ввод/вывод;

- поддержка параметров и флагов;

- доступность к внедрению в сторонние системы с интерфейсом и без;

- расширяемая система команд.

## 2.5 Сообщения системы

Система формирует пользовательские сообщения, отражающие состояние выполнения операций.

Основные типы сообщений:

1. **Информационные сообщения**

Используются при выводе справки и статуса:

```
${APP_NAME}$ - Document autofill tool
```

2. **Сообщения об успешном выполнении**

```
Table "students" created successfully
Template "resume" registered
Generated resume DOCX: output/resume.docx
```

3. **Ошибки выполнения**

```
Error: table not found
Error: template does not exist
Error: invalid command syntax
```

4. **Диагностические сообщения**

Используются при разработке и отладке:

```
Router: command parsed successfully
Database: connection initialized
Generate: template loaded
```

Сообщения системы спроектированы таким образом, чтобы обеспечивать:

- понятность для пользователя;

- минимальную избыточность;

- информативность при ошибках.

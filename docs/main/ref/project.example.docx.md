> Учебным планом направления 09.03.01 «Информатика и вычислительная
> техника» профиль: Искусственный интеллект, робототехника и сенсорика;
> по учебной дисциплине «Основы алгоритмизации и программирования»
> (далее ОАиП) предусмотрено выполнение курсового проекта.
>
> *Курсовой проект* -- учебная работа, содержащая результаты
> теоретических и (или) экспериментальных исследований по отдельной
> учебной дисциплине и включающая совокупность аналитических, расчетных,
> исследовательских, оценочных заданий, объединенных общностью
> рассматриваемого объекта и предполагающая выполнение отдельных
> элементов конструкторских, технологических, программных,
> организационно-управленческих, экономических и других работ и
> разработку графической документации, в том числе плакатов \[1\].
>
> **Целью** выполнения курсового проекта является углубление
> теоретических знаний и практических навыков в области алгоритмизации
> задач и разработки программных средств, развитие навыков
> самостоятельного изучения предметной области поставленной задачи.

>

## Требования к реализации курсового проекта:

- *Обязательным учебным элементом* курсового проекта по ОАиП является
  проектирование и реализация алгоритмов.

- В работе должны быть описаны и реализованы ***не менее 2** (двух)*
  алгоритмов*.*

- Вид программы - консольное приложение.

- Допустимые *языки программирования* - С++/C#

- Рекомендуемая среда разработки - *Microsoft Visual Studio (Visual
  Code)*. Допускается использование других IDE.

## ТРЕБОВАНИЯ К ПОЯСНИТЕЛЬНОЙ ЗАПИСКЕ

- *Объем* основной части пояснительной записки курсового проекта
  составляет **25-40 страниц А4**.

- Текст пояснительной записки оформляется в соответствии с требованиями
  оформления КП.

- Список используемых источников должен содержать ***не менее 5***
  (*пяти*) источников и оформляется в точном соответствии с правилами
  составления библиографического списка, установленными стандартом ГОСТ
  7.1-2003 \[5\].

- Текст программы (авторские фрагменты программного кода) обязательно
  приводится в приложении(-ях) к пояснительной записке. Текст программы
  должен быть документирован, это значит, что в программе нужно добавить
  комментарии и пояснения, которые объясняют, что делает конкретный
  кусок кода. Документированный код помогает другим разработчикам (или
  вам в будущем) понять логику, назначение и работу программы без
  необходимости разбирать каждую строчку.

Вот что обычно включает документирование программы:

- Комментарии внутри кода: краткие пояснения к функциям, переменным,
  алгоритмам.

- Описание функций и методов: что делает функция, какие аргументы
  принимает, что возвращает.

- Общие комментарии: описание целей программы, её структуры и логики.

- Документационные строки (докстринги): используют тройные кавычки или
  специальные форматы (например, для Python - \"\"\"Описание
  функции\"\"\"), чтобы автоматически генерировать документацию.

Пример:

#include \<iostream\>

/\*\*

\* \@brief Эта функция складывает два целых числа.

\*

\* Принимает два целых числа в качестве аргументов и возвращает их
сумму.

\*

\* \@param a Первое целое число.

\* \@param b Второе целое число.

\* \@return Сумма чисел a и b.

\*/

int add(int a, int b) {

return a + b;

}

/\*\*

\* \@brief Основная функция программы.

\*

\* В этой функции вызывается функция add для сложения двух чисел,

\* и результат выводится на экран.

\*

\* \@return int Код завершения программы, 0 означает успешное
выполнение.

\*/

int main() {

int num1 = 5; // Первое число для сложения

int num2 = 10; // Второе число для сложения

// Вызываем функцию add и сохраняем результат

int sum = add(num1, num2);

// Вывод результата на экран

std::cout \<\< \"Сумма \" \<\< num1 \<\< \" и \" \<\< num2 \<\< \" равна
\" \<\< sum \<\< std::endl;

return 0;

}

Что сделано:

- Используются комментарии в стиле Doxygen (/\*\* \... \*/), чтобы
  задокументировать функции и пояснить, что они делают.

- Внутри функций - комментарии для объяснения логики.

- Это делает код более понятным и легко поддерживаемым.

  - Графическая часть включает в себя схемы алгоритма и программы на
    листах ***формата А4*** в соответствии со стандартом ГОСТ 19.701-90
    \[6\].

**ПРИЛОЖЕНИЕ Д**

**Оценка сложности алгоритма**

Это анализ того, сколько ресурсов (время и память) требуется для
выполнения алгоритма при увеличении объема данных.

**Основные виды оценки:**

- Оценка по времени: время выполнения в зависимости от входных данных,
  обычно с помощью обозначений «Big O» (например, O(n), O(n\^2),
  O(log n) и т.д.).

<!-- -->

- Оценка по памяти: так же - сколько памяти потребуется в зависимости от
  размера данных.

Почему это важно?

Потому что, выбирая алгоритм, нужно учитывать:

- эффективность (сколько времени он займет);

- возможность масштабирования (работает ли он при больших объемах
  данных?);

- ограничения системы (например, памяти).

Пример: сортировка массива чисел

Рассмотрим два алгоритма сортировки:

**1. Bubble Sort (пузырьковая сортировка)**

Обоснование: понятный, простой для реализации, но очень медленный при
больших данных.

Сложность по времени: O(n\^2), где n - число элементов.

Подходит для: небольших массивов или для обучения.

**2. Быстрая сортировка (QuickSort)**

Обоснование: Быстрая и эффективная для средних и больших данных.

Сложность по времени: в среднем O(n log n), в худшем случае O(n\^2).

Подходит для: больших объемов данных, когда важна скорость.

Как провести оценку сложности?

Пример: Быстрая сортировка.

Обоснование:

В среднем, при разбиении массива, каждый раз делим его пополам, и так
повторяется log n раз (глубина рекурсии).

На каждом уровне обработки проверяются все элементы массива, что в
сумме - n элементов.

Поэтому суммарное время примерно O(n log n).

Худший случай: если массив уже отсортирован, и при этом алгоритм
работает как сортировка вставками - сложность может стать O(n\^2), но
при правильной реализации (например, с рандомизацией) это маловероятно.

документация и обоснование выбора алгоритма

cpp

#include \<vector\>

#include \<algorithm\> // для использования std::sort

#include \<iostream\>

/\*\*

\* \@brief Функция сортировки массива целых чисел.

\*

\* Используется алгоритм std::sort --- встроенная реализация быстрой
сортировки.

\* Она очень эффективна и обладает хорошей оценкой времени выполнения.

\*

\* \@param arr Ссылка на вектор целых чисел, который нужно
отсортировать.

\*

\* \@note

\* - В среднем время работы: O(n log n).

\* - В худшем случае: O(n\^2), но благодаря реализации с рандомизацией,

\* вероятностный случай практически исключен.

\*/

std::sort(arr.begin(), arr.end());

}

/\*\*

\* \@brief Основная функция программы.

\*

\* Создает массив, сортирует его и выводит результат.

\* Проведен анализ сложности алгоритма сортировки.

\*/

int main() {

std::vector\<int\> data = {5, 2, 8, 1, 3, 9, 4};

// Обоснование выбора алгоритма

// Используем std::sort, потому что:

// - Быстрая сортировка внутри она обеспечит быстрое выполнение

// - Средняя сложность O(n log n), что подходит для больших данных

// - Имеется встроенная оптимизация и рандомизация для избегания худших
случаев

sortArray(data);

// Вывод отсортированных данных

for (int num : data) {

std::cout \<\< num \<\< \" \";

}

std::cout \<\< std::endl;

return 0;

}
